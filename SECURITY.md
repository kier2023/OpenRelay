# Security Policy

## Supported Versions

Security updates are provided for the latest stable version of this project.

Because the project is maintained by a single developer, older versions are not actively maintained. Users are encouraged to upgrade to the latest release whenever possible.

| Version | Supported |
| ------- | --------- |
| Latest release | ✅ |
| Older releases | ❌ |

If you are running an older version, upgrading to the latest version is the recommended mitigation for most issues.

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

Do **not** open a public issue for security vulnerabilities.

Instead, report the issue privately through one of the following methods:

- GitHub Security Advisories (preferred)
- Direct contact through the repository maintainer

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps required to reproduce the vulnerability
- The potential impact of the issue
- Any suggested fixes or mitigations (if known)

Providing clear reproduction steps significantly improves the chances of a quick resolution.

---

## Response Process

After receiving a report, the maintainer will:

1. Review the vulnerability report
2. Attempt to reproduce the issue
3. Determine the severity and impact
4. Prepare a fix if the vulnerability is confirmed

If the issue is valid, a patch will be prepared and released as soon as reasonably possible.

---

## Disclosure

Responsible disclosure is appreciated.

Once a fix has been released, the vulnerability may be publicly documented in the project's security advisory or release notes.

This helps users understand the risk and encourages upgrading to secure versions.

---

## Scope

This policy applies only to vulnerabilities in the project's source code.

Issues caused by misconfiguration, third-party services, or external platforms (such as Twitter/X, Telegram, Discord, or other APIs) are outside the scope of this security policy.
